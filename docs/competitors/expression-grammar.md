### Сравнение синтаксической структуры C# и C++ с упором на числовые типы данных

### 1. Сравнение числовых типов данных и их характеристик

**C++**:
- Целочисленные: `char`, `short`, `int`, `long`, `long long` (размер зависит от платформы).
- Беззнаковые: `unsigned char`, `unsigned int` и т.д.
- Вещественные: `float`, `double`, `long double`.
- Фиксированный размер: `int32_t`, `uint64_t` (из `<cstdint>`).
- Размер типа можно узнать через `sizeof(type)`, но он варьируется.

**C#**:
- Целочисленные: `sbyte`, `short`, `int`, `long`, `byte`, `ushort`, `uint`, `ulong` (строго фиксированный размер).
- Вещественные: `float`, `double`, `decimal` (для финансовых расчётов).
- Все типы — структуры (например, `int` — псевдоним для `System.Int32`).
- Типы с платформо-зависимым размером: `nint`, `unint` (для взаимодействия с нативным кодом).

**Ключевые различия**:
- В C# размер типов фиксирован, в C++ — зависит от платформы (например, `int` может быть 16, 32 или 64 бита).
- В C# нет беззнаковых типов по умолчанию (кроме `byte`), но они доступны явно.
- Тип `decimal` в C# отсутствует в C++.
- В C++ `sizeof(char)` всегда 1, но `sizeof(int)` не фиксирован. В C# `sizeof(int)` всегда 4, но для непримитивных типов требует небезопасного контекста (`unsafe`).

### 2. Сравнение арифметических операторов и обработки переполнений

**Общие операторы**: `+`, `-`, `*`, `/`, `%`, `++`, `--`.

**C++**:
- Оператор `%` применим только к целым числам.
- Перегрузка операторов разрешена для пользовательских типов.
- Деление целых чисел: результат целый (округление в сторону нуля).
- Переполнение целочисленных типов — **Неопределенное Поведение (UB)**. Результат непредсказуем и зависит от компилятора и платформы.

**C#**:
- Оператор `%` работает и с вещественными числами.
- Перегрузка операторов возможна через статические методы.
- Деление целых: как в C++, но для вещественных — точное.
Контроль переполнений через контексты `checked` и `unchecked`:
  - По умолчанию (`unchecked`): переполнение оборачивается (например, `int.MaxValue + 1 == int.MinValue`).
  - В контексте `checked`: генерируется `OverflowException`.

**Пример переполнения**:
```cpp
// C++ (Неопределенное Поведение!)
int max = INT_MAX;
int result = max + 1; // Результат непредсказуем!
```
```csharp
// C#
int max = int.MaxValue;
int uncheckedResult = unchecked(max + 1); // -2147483648 (корректное оборачивание)
checked {
    int checkedResult = max + 1; // Выбрасывает OverflowException
}
```

### 3. Сравнение перечислений (enum)
**C++**:
- Не типобезопасны: неявно преобразуются в `int` и между собой.
- Размер зависит от компилятора.
```cpp
enum Color { Red, Green, Blue };
enum TrafficLight { Yellow, Red }; // Ошибка: Red переопределен!
int x = Red; // Легально, неявное преобразование
```

**C#**:
- Типобезопасны: требуют явного приведения.
- По умолчанию базовый тип — `int`, но можно изменить.
```csharp
enum Color { Red, Green, Blue }
// int x = Color.Red; // Ошибка: требуется приведение
int x = (int)Color.Red; // Легально
// Color color = 1; // Ошибка: требуется приведение
Color color = (Color)1; // Легально
```

### 4. Сравнение приоритетов операций

| Приоритет | Категория               | Операторы                                   | Ассоциативность |
|-----------|-------------------------|-------------------------------------        |-----------------|
| 1         | Группировка, вызов      | `()`, `[]`, `.`, `->` (C++), `::` (C++)     | Слева направо   |
| 2         | Унарные                 | `!`, `~`, `+`, `-`, `++`, `--`, `(тип)`, `*`, `&`, `sizeof` (C++), `new`, `typeof` (C#) | Справа налево |
| 3         | Мультипликативные       | `*`, `/`, `%`                               | Слева направо   |
| 4         | Аддитивные              | `+`, `-`                                    | Слева направо   |
| 5         | Сдвиги                  | `<<`, `>>`                                  | Слева направо   |
| 6         | Сравнение               | `<`, `>`, `<=`, `>=`, `is` (C#), `as` (C#)  | Слева направо   |
| 7         | Равенство               | `==`, `!=`                                  | Слева направо   |
| 8         | Побитовое AND           | `&`                                         | Слева направо   |
| 9         | Побитовое XOR           | `^`                                         | Слева направо   |
| 10        | Побитовое OR            | `\|`                                        | Слева направо   |
| 11        | Логическое AND          | `&&`                                        | Слева направо   |
| 12        | Логическое OR           | `\|\|`                                      | Слева направо   |
| 13        | Условный оператор       | `?:`                                        | Справа налево   |
| 14        | Присваивание            | `=`, `+=`, `-=`, `*=`, `/=`, `%=`, `<<=`, `>>=`, `&=`, `^=`, `\|=` | Справа налево |
| 15        | Запятая                 | `,`                                         | Слева направо   |


#### **Ключевые различия**

##### **C++:**
1. **Оператор `::`** (разрешение области видимости) — приоритет 1.
2. **Операторы `.*` и `->***` (указатели на члены классов)** — приоритет 4 (после `->` и `.`).
3. **`noexcept`** — приоритет 2 (унарный).
4. **Приведение в стиле C** `(тип)` — приоритет 2.
5. **`sizeof`**, `alignof` — приоритет 2.

##### **C#:**
1. **`is` и `as`** — приоритет 6 (как сравнение).
2. **`??` (null-объединение)** — приоритет 13 (после `?:`).
3. **`=>` (лямбда)** — низкий приоритет (не включен в общую таблицу).
4. **`await`** — приоритет 2 (унарный).
5. **`nameof`**, `default` — приоритет 1 (первичные выражения).
6. **`?.` и `?[]` (null-условие)** — приоритет 1 (как `.` и `[]`).

### 5. Сравнение встроенных функций и констант времени компиляции

**C++**:
- Библиотека `<cmath>`: `sqrt()`, `sin()`, `pow()`, `abs()`.
- Библиотека `<algorithm>`: `std::max()`, `std::min()`.
- Для целых: `std::gcd()`, `std::lcm()` (C++17).
- `constexpr` для вычислений во время компиляции:
  
```cpp
constexpr double sqrt2 = std::sqrt(2.0); // Вычисляется на этапе компиляции
```

**C#**:
- Класс `Math`: `Math.Sqrt()`, `Math.Sin()`, `Math.Pow()`, `Math.Abs()`.
- `Math.Max()`, `Math.Min()`.
- Для целых: `Math.BigMul()` (для больших чисел).
- Статические методы в типах: `int.Parse()`, `double.TryParse()`.
- Константы времени компиляции через `const` (только для простых типов):
  
```csharp
const double Pi = 3.14159; // Константа компиляции
// const double sqrt2 = Math.Sqrt(2.0); // Ошибка: Math.Sqrt нельзя в const
```
