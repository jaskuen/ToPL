## Компиляция

Код на C++ преобразуется в машинный код **сразу** после компиляции.
В C# код сначала преобразуется в промежуточный язык (IL), а затем уже в машинный код.

## Объявление переменных

### C++

```cpp
// Явное указание типа
int x = 5;
double y = 3.14;
const char* str = "Hello";

// Автовывод типа (C++11)
auto a = 5; // int
auto b = 3.14; // double

// Указатели и ссылки
int* ptr = &x;
int& ref = x;
```

### C\#

```cs
// Явное указание типа
int x = 5;
double y = 3.14;
string str = "Hello";

// Неявное указание типа (C# 3.0)
var a = 5; // int
var b = 3.14; // double

// Ссылочные типы (управляемые указатели)
object obj = new object();
```

**Ключевые различия**:
- C++ поддерживает сырые указатели, C# — только управляемые ссылки
- C++ `auto` vs C# `var` (разная семантика выведения типа)
- В C++ константность через `const`, в C# через `readonly`

### Блоки кода и области видимости

Оба языка используют `{}`, но:
- В C++ область видимости определяется блоком
- В C# область видимости определяется блоком + правила CLR
## Операторы

### Арифметические

Оба языка поддерживают: `+ - * / % ++ --`
### Логические

Оба языка: `&& || !` (C++ также поддерживает `and or not`)
### Сравнения

Оба языка: `== != < > <= >=`
### Особые операторы

```cpp
:: // область видимости
.* ->* // указатели на члены
```

```cs
as // безопасное приведение типов
is // проверка типа
?? // null-coalescing
?. // null-conditional
```

## Обработка ошибок

Оба языка имеют блоки `try`, `catch`. 
Но C# имеет блок `finally` - он гарантирует, что код, находящийся в нем запустится, даже если возникло исключение в блоке `try`. 

## Структуры

В C++ структура по сути является классом, в котором все члены по умолчанию являются публичными. В структуре C++ можно определять функции-члены, конструкторы и деструкторы — точно так же, как в классе.
Структуры в C++ являются типами значений и обычно размещаются в стеке, если явно не размещаются в куче с помощью оператора new.
```cpp
struct Point 
{
    double x, y;
    Point(double x, double y) : x(x), y(y) {}
    
    double magnitude() const 
    {
        return sqrt(x*x + y*y);
    }
};

Point p(1.0, 2.0);
```

В C# структура является типом значения, в отличие от класса, который является ссылочным типом. Они обычно размещаются в стеке и передаются по значению, что означает создание копии при передаче в метод.
Структуры в C# имеют ограничения по сравнению с классами: они не могут наследовать от других классов (хотя могут реализовывать интерфейсы), у них не может быть конструктора по умолчанию без параметров, и их поля не могут быть инициализированы напрямую при объявлении.

## Управляющие конструкции
Оба языка поддерживают циклы `for`, `while`, `do-while`.
Ветвления `if`, `else`, `switch`, `case`. Однако, в C++ в `switch`допускает fall-through, в C# требуется явный `break`.

Также есть итерируемые циклы. В C# они реализованы через `foreach` и перечисления. 
```cs
foreach (var item in collection)
{
	// . . .
}
```

В C++ используется следующая семантика:
```cpp
for (auto item : array)
{
	// . . .
}
```
Но, он работает только с итераторами `begin()` и `end()`. Его более продвинутая версия - `for_each` может работать с произвольным диапазоном данных и его можно сделать многопоточным.

## Управление памятью

В C++ управление памятью происходит в ручном режиме, в C# для этого есть `garbage collector` - он отследит, когда объект завершит задачу и уничтожит его.
Дополнительно, в C++ есть умные указатели, которые отчасти решают проблемы работы с сырыми указателями.

### C++
```cpp
// Стек (автоматическое)
void func() 
{
    int x = 10;          // На стеке
    Point p{ 1, 2 };       // На стеке
}

// Куча (ручное управление)
int* x = new int(10);    // Выделение
delete x;                // Освобождение

// Массивы
int* arr = new int[100];
delete[] arr;
```

### C\#

```cs
// Стек (value types)
void Method() 
{
    int x = 10;         // На стеке
    Point p = new Point(1, 2); // На стеке (struct)
}

// Управляемая куча
object obj = new object(); // Выделение
// Освобождение автоматическое

// Массивы
int[] arr = new int[100]; // Управляемый массив
```

## Функции и методы
В C++ есть свободные функции - их область видимости - текущее пространство имен или блок. В C# есть только методы, аналогом свободной функции является статическая функция.

### C++
```cpp
// Возвращаемый_тип имя_функции(параметры)
int add(int a, int b) 
{
    return a + b;
}

// Const метод (не изменяет состояние объекта)
class Calculator {
    int value;
public:
    int getValue() const { return value; } // const метод
    void setValue(int v) { value = v; }    // не-const метод
};
```

### C\#
```cs
// [модификаторы] возвращаемый_тип имя_метода(параметры)
public int Add(int a, int b) {
    return a + b;
}

// Методы с модификаторами доступа
class Calculator {
    private int value;
    
    public int GetValue() { return value; }     // только чтение
    public void SetValue(int v) { value = v; }  // изменение
}
```

## Передача параметров

### C++
```cpp
void processValues(int byValue,      // По значению (копия)
                  int& byReference,  // По ссылке (изменяемый)
                  const int& byConstRef)  // По const ссылке - неизменяемый
{
    byValue = 10;    // Изменяет копию
    byReference = 20; // Изменяет оригинал
}

// Указатели
void modifyPointer(int* ptr) 
{
    if (ptr) *ptr = 100; // Разыменование и изменение
}
```

### C\#
```cs
void ProcessValues(int byValue,      // Value type по значению
                  ref int byRef,     // По ссылке
                  in int byIn,       // Readonly ссылка (C# 7.2)
                  out int byOut) {   // Выходной параметр
    byValue = 10;    // Изменяет копию
    byRef = 20;      // Изменяет оригинал
    // byIn = 30;    // Ошибка компиляции
    byOut = 40;      // Инициализация выходного параметра
}

// Использование ref, in, out
int x = 1, y, z;
ProcessValues(x, ref x, in x, out y);
```
