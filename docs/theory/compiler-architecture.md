## Чем отличаются компилятор и интерпретатор?

**Компилятор** — это программа-транслятор, которая **полностью преобразует** исходный код программы на языке высокого уровня в **машинный код** (или в промежуточное представление), пригодное для непосредственного выполнения целевым процессором или виртуальной машиной.  
Процесс происходит **до запуска программы** (этап компиляции), а результатом является **исполняемый файл**.

**Интерпретатор** — это программа, которая **выполняет исходный код напрямую**, без предварительного полного перевода в машинный код.  
Он **анализирует и выполняет** программу **построчно** (или по инструкциям, покомандно), обычно в режиме реального времени.

> **Простой интерпретатор** — считывает, анализирует и сразу выполняет каждую строку/инструкцию.
>
> **Интерпретатор компилирующего типа** — сначала компилирует исходный код в **байт-код** (промежуточное представление), а затем интерпретирует этот байт-код с помощью **виртуальной машины** (например, JVM для Java или CPython для Python).

**Транслятор** — общее название для программ, преобразующих код с одного языка на другой.  
Компиляторы и интерпретаторы — это **разновидности трансляторов**.

> **Цель трансляции** — сделать программу понятной **адресату**:
>
> - Если адресат — **процессор**, то трансляция называется **компиляцией в машинный код**.
>
> - Если адресат — **интерпретатор или виртуальная машина**, то трансляция может называться **интерпретацией** или **компиляцией в байт-код**.

## Преимущества компилятора перед интерпретатором

1. **Быстродействие** — скомпилированный машинный код выполняется непосредственно процессором без дополнительных затрат на анализ и интерпретацию.
2. **Оптимизация** — компилятор может проводить глубокую оптимизацию кода на этапе компиляции.
3. **Раннее обнаружение ошибок** — большинство ошибок выявляются **до запуска программы**.
4. **Безопасность и защита интеллектуальной собственности** — для запуска программы не требуется исходный код; распространяется только двоичный файл.
5. **Независимость от среды выполнения** — для запуска не нужен компилятор или интерпретатор.

## Чем компиляция в машинный код отличается от компиляции в байт-код?

>  Из-за необходимости интерпретации байт-код выполняется __значительно медленнее__ машинного кода сравнимой функциональности, но такой код __более переносим__ ( не зависит от ОС и модели процессора ).
>
>  Чтобы __ускорить__ выполнение байт-кода, используется _динамическая компиляция_, когда виртуальная машина _транслирует_ псевдокод в машинный код непосредственно перед его первым исполнением ( Как пример - JIT компилятор )
