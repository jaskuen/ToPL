# `W Language` – Семантическая спецификация

Документ описывает смысл языковых конструкций, систему типов и правила выполнения программы.

## 1. Система типов

Язык статически типизирован. Все проверки типов выполняются на этапе компиляции.

### 1.1 Базовые типы

| Тип      | Аналог .NET      | Диапазон / Описание                                       | По умолчанию |
| -------- | ---------------- | --------------------------------------------------------- | ------------ |
| `int`    | `System.Int32`   | −2 147 483 648 – 2 147 483 647 (32-битное знаковое целое) | `0`          |
| `float`  | `System.Single`  | ±1.5 × 10⁻⁴⁵ – ±3.4 × 10³⁸ (32-битное, 7 значащих цифр)   | `0.0`        |
| `string` | `System.String`  | Неизменяемая строка UTF-8                                 | `""`         |
| `bool`   | `System.Boolean` | `true` или `false`                                        | `false`      |
| `void`   | `System.Void`    | Отсутствие значения (только для функций)                  | –            |

>[!INFO]
> - `bool` – отдельный тип, не приводится к числам.
> - `true` и `false` – единственные допустимые значения типа `bool`.
> - Для `float` поддерживается как +0.0, так и -0.0.

### 1.2 Приведение типов

**Неявное приведение:**

| Из       | В        | Разрешено                              |
| -------- | -------- | -------------------------------------- |
| `int`    | `float`  | Да ( Возможна потеря точности )        |
| `int`    | `string` | Да ( Только в `write` и конкатенации ) |
| `float`  | `string` | Да ( Только в `write` и конкатенации ) |
| `float`  | `int`    | Нет                                    |
| `string` | `int`    | Нет                                    |
| `string` | `float`  | Нет                                    |
| `bool`   | любой    | Нет                                    |
| любой    | `bool`   | Нет                                    |

>[!TIP]
> Явное приведение отсутствует в текущей версии языка `W Language`.

### 1.3 Операции с типами

| Операция         | `int` | `float` | `string` | `bool` |
| ---------------- | ----- | ------- | -------- | ------ |
| `+`              | Да    | Да      | Да       | Нет    |
| `-` (бинарный)   | Да    | Да      | Нет      | Нет    |
| `-` (унарный)    | Да    | Да      | Нет      | Нет    |
| `*`              | Да    | Да      | Нет      | Нет    |
| `/`              | Да    | Да      | Нет      | Нет    |
| `%`              | Да    | Нет     | Нет      | Нет    |
| `++` (префикс)   | Да    | Да      | Нет      | Нет    |
| `++` (постфикс)  | Да    | Да      | Нет      | Нет    |
| `--` (префикс)   | Да    | Да      | Нет      | Нет    |
| `--` (постфикс)  | Да    | Да      | Нет      | Нет    |
| `==`, `!=`       | Да    | Да      | Да       | Да     |
| `<`, `<=`        | Да    | Да      | Да       | Нет    |
| `>`, `>=`        | Да    | Да      | Да       | Нет    |
| `&&`, `and`      | Нет   | Нет     | Нет      | Да     |
| `\|\|`, `or`     | Нет   | Нет     | Нет      | Да     |
| `!`, `not`       | Нет   | Нет     | Нет      | Да     |

>[!TIP]
> `+` для строк работает как конкатенация. Если один из операндов – строка, второй неявно приводится к строке.
> Сравнение строк выполняется лексикографически( по кодам UTF-8 ), регистрозависимо.

>[!WARNING]
> Деление на ноль вызывает ошибку времени выполнения.
> Операция `%` также вызывает ошибку времени выполнения при нулевом делителе.
> Конкатенация возможна только между двумя `string`.
> Операндами `++` / `--` должны быть __изменяемые переменные__.

## 2. Область видимости и идентификаторы

### 2.1 Область видимости

**Лексическая область видимости:** Переменная видима внутри блока `{}`, в котором объявлена, и во всех вложенных блоках.
**Глобальная область:** Переменные и функции, объявленные вне функций, видны во всём файле.
**Локальная область:** Переменные внутри функции видны только в этой функции.

### 2.2 Уникальность имён
>[!INFO]
> В пределах одного блока имена переменных должны быть уникальны.
> Внутренняя переменная может скрывать внешнюю с тем же именем.
> Имена функций должны быть уникальны в рамках программы (перегрузка не поддерживается).
> Переменные и функции находятся в **разных пространствах имён** (можно назвать переменную так же, как функцию, но не рекомендуется).

### 2.3 Ограничения
>[!INFO]
> **Нельзя** скрыть ключевое слово именем переменной или функции.
> **Нельзя** скрыть имя встроенной функции (`read`, `write`).
> **Нельзя** скрыть имя типа данных (`int`, `float`, `string`, `bool`, `void`).

## 3. Переменные и константы

### 3.1 Переменные

Язык позволяет объявлять статически-типизированные переменные с указанием типа, который не может быть изменен в процессе выполнения программы.

**Объявление:**
```cpp
int x;
int y = 10;
```

__Множественное объявление:__
```cpp
int a, b, c;
int x = 1, y = 2, z;
float pi = 3.14, e = 2.71;
```

>[!INFO]
> Должны быть объявлены перед использованием.
> Если инициализация не указана, присваивается значение по умолчанию для типа.
> Можно присваивать новое значение многократно.
> Тип не может быть изменён после объявления.
> Имена переменных не могут совпадать с ключевыми словами.
> В множественном объявлении инициализаторы вычисляются слева направо. Допускается использовать ранее объявленные переменные из этого же списка.
> Каждый элемент списка обязан иметь инициализатор.

>[!WARNING]
> Язык не поддерживает создание динамически-типизированных переменных с помощью ключевых слов `let` или `var`.

### 3.2 Константы

Для создания неизменяемой переменной используется ключевое слово `const`.

**Объявление:**
```cpp
const int MAX = 100;
const int MIN = 0, DEF = 50;
```

>[!INFO]
> Обязаны быть инициализированы в момент объявления.
> Попытка присвоения после инициализации вызывает ошибку компиляции.
> Операторы `++` и `--` не применимы к константам.

## 4. Функции

### 4.1 Объявление

Для объявления пользовательской функции требуется указать знаки `(` и `)` после названия функции.

**Синтаксис:**
```cpp
type identifier(parameter_list) { ... }
```

**Пример:**
```cpp
int Square(int x) {
    return x * x;
}
```

>[!INFO]
> Функция __обязана__ иметь возвращаемый тип (включая `void`).
> Имя функции должно быть уникальным в программе.
> Параметры передаются по значению.
> Функции могут быть объявлены только глобально (вложенные функции не поддерживаются).
> Функция не может выполнять неявные преобразования типов при возврате значений.
> Если функция не имеет возвращаемого значения, используется ключевое слово `void`.

### 4.2 Точка входа
>[!INFO]
> Функция `main` выполняется первой.
> Возвращаемый тип: `void` или `int`.
> `int main()` возвращает код завершения программы (0 – успех).
> `void main()` автоматически завершается с кодом 0.

>[!TIP]
> Аргументы не поддерживаются в текущей версии.

### 4.3 Параметры
>[!INFO]
> Передаются **по значению**.
> Изменение параметра внутри функции не влияет на аргумент.
> Число формальных и фактических параметров должно совпадать.
> Типы параметров должны совпадать (с учётом неявного приведения).

### 4.4 Возврат значения
>[!INFO]
> Если указан тип возврата (не `void`), **каждый возможный путь выполнения** функции обязан заканчиваться инструкцией `return` с выражением, тип которого совпадает с объявленным (с учётом неявных приведений). Иначе ошибка компиляции.
> Если тип возврата `void`, `return` может использоваться без аргумента для досрочного выхода; использование `return` с выражением любого типа запрещено.
> Анализ достижимости `return` выполняется статически.


### 4.5 Рекурсия
>[!INFO]
> **Прямая рекурсия:** Поддерживается (функция может вызывать саму себя).
> **Взаимная рекурсия:** Поддерживается при условии объявления функций до использования.

### 4.6 Встроенные функции

Язык предоставляет набор встроенных функций для работы с числами и строками:

| Функция                       | Описание                               | Типы параметров    | Возвращаемый тип |
| ----------------------------- | -------------------------------------- | ------------------ | ---------------- |
| `abs(x)`                      | Модуль числа                           | `int` или `float`  | Тот же тип       |
| `round(x)`                    | Математическое округление              | `float`            | `int`            |
| `ceil(x)`                     | Округление вверх                       | `float`            | `int`            |
| `floor(x)`                    | Округление вниз                        | `float`            | `int`            |
| `min(x, y)`                   | Минимальное из двух значений           | `int` или `float`  | Тот же тип       |
| `max(x, y)`                   | Максимальное из двух значений          | `int` или `float`  | Тот же тип       |
| `length(s)`                   | Длина строки                           | `string`           | `int`            |
| `substring(s, start, length)` | Получение подстроки по индексу и длине | `string, int, int` | `string`         |
>[!INFO]
> Встроенные функции не могут быть переопределены пользователем.
> Типы параметров должны строго соответствовать сигнатуре функции.
> Для `min` и `max` оба параметра должны быть одного типа (`int` или `float`).
> Если `start` или `length` выходят за границы строки, возникает ошибка времени выполнения.

## 5. Ветвления и циклы

### 5.1 Ветвление `if`

Язык поддерживает ветвления с помощью ключевых слов `if` и `else`.

**Синтаксис:**
```cpp
if (условие) {
} else {
}
```

>[!TIP]
> Блок `else` является необязательным.

### 5.2 Цикл `while`

Язык поддерживает циклы с помощью ключевого слова `while`.

**Синтаксис:**
```cpp
while (условие) {
}
```

### 5.3 Инструкции управления циклом

| Инструкция | Описание                                      | Ограничения        |
| ---------- | --------------------------------------------- | ------------------ |
| `break`    | Немедленно завершает ближайший объемлющий цикл | Только внутри цикла |
| `continue` | Переходит к проверке условия цикла            | Только внутри цикла |

>[!WARNING]
> Использование `break` и `continue` вне цикла запрещено (ошибка компиляции).

## 6. Ввод и вывод

Язык поддерживает ввод и вывод данных через консоль с помощью ключевых слов `read` и `write`.

### 6.1 Вывод `write`

**Синтаксис:**
```cpp
write(выражение);
```

>[!INFO]
> Выводит значение выражения в стандартный поток вывода.
> Поддерживает типы `int`, `float`, `string`, `bool`.
> **Не добавляет** символ новой строки в конце.
> Для перехода на новую строку используйте `\n` в строке.
> Для реализации `writeLn` используйте escape-последовательность: `write("текст\n");`.
> Вызов `write()` без аргументов запрещен.

### 6.2 Ввод `read`

**Синтаксис:**
```cpp
read(переменная);
read();
```

>[!INFO]
> Считывает значение из стандартного потока ввода.
> Тип переменной определяет формат ввода.
> Поддерживает типы `int`, `float`, `string`.
> Для `bool` ввод не поддерживается в текущей версии.
> Если переменная не указана, `read()` считывает и игнорирует ввод.
> Для реализации `readLn` или `writeLn` используйте escape-последовательность.


## 7. Операторы и выражения

>[!INFO]
> - Все операторы регистрозависимы.
> - Логические операторы `&&` и `||` используют короткую схему вычисления (short-circuit).
> - Деление на ноль вызывает ошибку времени выполнения.

### 7.1 Арифметические операторы

**Бинарные операторы:**
- `+` — сложение (`int`, `float`) или конкатенация, если хотя бы один операнд - строка (`string`).
- `-` — вычитание (`int`, `float`).
- `*` — умножение (`int`, `float`).
- `/` — деление (`int`, `float`).
- `%` — остаток от деления (только `int`).

**Унарные операторы:**
- `-` — унарный минус (`int`, `float`).
- `++` (префикс) — префиксный инкремент (`int`, `float`): увеличивает значение, затем возвращает его.
- `++` (постфикс) — постфиксный инкремент (`int`, `float`): возвращает значение, затем увеличивает его.
- `--` (префикс) — префиксный декремент (`int`, `float`): уменьшает значение, затем возвращает его.
- `--` (постфикс) — постфиксный декремент (`int`, `float`): возвращает значение, затем уменьшает его.

**Примеры:**
```cpp
int x = 5;
int a = ++x;
int b = x++;
```

>[!INFO]
> При смешении `int` и `float` результат приводится к `float`.
> Конкатенация `+` работает только если оба операнда типа `string`.
> Операторы `++` и `--` не применимы к константам.
> Деление на ноль вызывает ошибку времени выполнения.

**Таблица совместимости типов:**

| Оператор       | `int` | `float` | `string` | Результат           |
| -------------- | ----- | ------- | -------- | ------------------- |
| `+`            | Да    | Да      | Да       | `int`/`float`/`string` |
| `-` (бинарный) | Да    | Да      | Нет      | `int`/`float`       |
| `-` (унарный)  | Да    | Да      | Нет      | `int`/`float`       |
| `*`            | Да    | Да      | Нет      | `int`/`float`       |
| `/`            | Да    | Да      | Нет      | `int`/`float`       |
| `%`            | Да    | Нет     | Нет      | `int`               |
| `++` (префикс) | Да    | Да      | Нет      | `int`/`float`       |
| `++` (постфикс)| Да    | Да      | Нет      | `int`/`float`       |
| `--` (префикс) | Да    | Да      | Нет      | `int`/`float`       |
| `--` (постфикс)| Да    | Да      | Нет      | `int`/`float`       |

### 7.2 Операторы сравнения

 `==`, `equals` — равенство.
 `!=` — неравенство.
 `<` — меньше.
 `<=` — меньше или равно.
 `>` — больше.
 `>=` — больше или равно.

>[!INFO]
> Все операторы сравнения возвращают тип `bool`.
> Сравнение возможно для типов `int`, `float`, `string`, `bool`.
> Для строк сравнение выполняется лексикографически (по кодам символов UTF-8).

### 7.3 Логические операторы

 `&&`, `and` — логическое И.
 `||`, `or` — логическое ИЛИ.
 `!`, `not` — логическое НЕ.

>[!INFO]
> Работают только с типом `bool`.
> Операторы `&&` и `||` используют короткую схему вычисления (short-circuit evaluation).
> Словесные (`and`, `or`, `not`) и символьные (`&&`, `||`, `!`) формы эквивалентны.
### 7.4 Оператор присваивания

 `=` — оператор присваивания.

>[!INFO]
> Левая часть должна быть изменяемой переменной (не константой).
> Тип правой части должен совпадать с типом левой части (с учётом неявного приведения).
> Присваивание возвращает присвоенное значение и может использоваться в выражениях.
> Присваивание имеет правую ассоциативность: `a = b = c` эквивалентно `a = (b = c)`.

## 8. Ограничения и ошибки

### 8.1 Инициализация

>[!INFO]
> Использование неинициализированной локальной переменной **разрешено**(присваивается значение по умолчанию).
> Глобальные переменные всегда инициализируются значениями по умолчанию.

### 8.2 Типизация

>[!INFO]
> Несовпадение типов в присваивании вызывает ошибку компиляции.
> Несовпадение типов в возврате функции вызывает ошибку компиляции.
> Неявное приведение `float` к `int` запрещено.

## 9. Дополнительные семантические правила

### 9.1 Порядок вычисления

>[!INFO]
> Аргументы функций вычисляются слева направо.
> Операнды бинарных операторов вычисляются слева направо.
> Логические операторы `&&` и `||` используют короткую схему вычисления.

### 9.2 Время жизни переменных

>[!INFO]
> Глобальные переменные существуют в течение всего времени выполнения программы.
> Локальные переменные создаются при входе в блок и уничтожаются при выходе.
> Параметры функций существуют только во время выполнения функции.

### 9.3 Правила для левой части присваивания

>[!INFO]
> В левой части присваивания может находиться только:
> - Идентификатор изменяемой переменной (не константы).
> - Нельзя присваивать значение литералу, результату выражения или константе.

### 9.4 Пустой оператор

>[!INFO]
> Пустой оператор `;` не выполняет никаких действий. Он допустим в любом месте, где ожидается инструкция, и не влияет на поток выполнения или проверку типов.
